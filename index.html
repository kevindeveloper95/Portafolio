Ritual Roast · Architecture evolution on AWS

Application with a Next.js frontend and a Flask backend. All AWS infrastructure is provisioned and automated with Terraform, so each version can be deployed, evolved and reproduced in a controlled way.

  • v1 · Three-tier architecture on EC2: Baseline with a multi-AZ VPC, ALB, ASG, EC2, RDS MySQL Multi-AZ, NAT Gateway, Secrets Manager and IAM — separating DMZ, application and data tiers with dedicated security groups.
  • v2 · Containers on ECS Fargate: Workloads containerized on ECS Fargate with images in ECR, ALB target groups and RDS Multi-AZ — removing direct server management and packaging Next.js and Flask as independent services with consistent deployments.
  • v3 · Serverless architecture: CloudFront and S3 for the static frontend, API Gateway as the entry point, Lambda functions for business logic and DynamoDB as the data store — minimizing infrastructure operations and scaling purely on demand.
  • Governance · Multi-account governance: AWS Organizations with management, development and production accounts, SCPs and cross-account IAM roles — enforcing centralized controls, security and compliance as the architecture grows.
